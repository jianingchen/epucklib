void el_camera_lock_frame ( )

lock the frame buffers

When the frame buffer is locked, no buffer swapping will occur. This guarantees the image inside the frame buffer to be one frame of image. Otherwise, the image may be changed halfway through a image processing algorithm.

void el_config_camera ( const el_camera_param *  p)

apply the parameters to the camera

Parameters
ppointer to the data structure

The default parameters of the camera is equivalent to the following code:

1 el_camera_param *CameraSetting;
2 ...
3 CameraSetting = el_config_camera_list();
4 CameraSetting->ExposureMode = EL_AUTOMATIC;
5 CameraSetting->AutoWhiteBalance = true;
6 CameraSetting->AutoDigitalGain = true;
7 CameraSetting->ExposureTime = 1.0f;
8 CameraSetting->RedGain = 1.0f;
9 CameraSetting->GreenGain = 1.0f;
10 CameraSetting->BlueGain = 1.0f;
11 CameraSetting->ExternalIntergationTime = 1.0f;
12 CameraSetting->ExternalLinearGain = 1.0f;
13 CameraSetting->IntegrationTime = 1.0f;
14 CameraSetting->GlobalGain = 1.0f;
15 CameraSetting->DigitalGain = 1.0f;
16 el_config_camera( CameraSetting );

The time cost to execute this function is not neglectable (around 7.1 ms), so it should not be executed frequently in a process.
